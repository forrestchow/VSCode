# 电商数据获取 — 项目规则

## 文档同步（核心规则）

每完成一个功能模块后，必须执行以下步骤：

1. **重读架构文档** — `docs/ARCHITECTURE.md` 是项目唯一真相源
2. **对比实现与文档** — 检查本次代码改动是否与文档描述一致
3. **同步差异**：
   - 代码是对的，文档落后了 → **更新文档**，让文档追上代码
   - 文档是对的，代码写偏了 → **修改代码**，让代码对齐文档
   - 两端都对，但设计变了 → **两端一起改**，保持一致
4. **一起提交** — 文档更新和代码改动放在同一个 git commit

> 简单说：代码和文档谁落后就改谁，始终保持二者一致。

## 技术约束

- **纯 Python + CDP**，不依赖 ChromeDriver、Selenium、浏览器插件
- **察尔汗 Chrome 访客**：`--remote-debugging-port=9222`，使用真实 profile 复用登录态
- **数据流**：`Runtime.evaluate` 注入 JS → 直接在 Python 拿返回值 → pandas 清洗 → PostgreSQL
- **不要引入新的通信层**（HTTP Server、WebSocket 等都不需要，CDP 一条线搞定）

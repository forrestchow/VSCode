## Step 1: 数据采集

### 计划

用 pynput 录制正常桌面鼠标轨迹。窗口内画布交互：随机起点/终点，点击切段，每条轨迹记录为 `[x, y, t_ms]` 序列，存入 JSONL。

### 文件

| 文件 | 用途 |
|------|------|
| `data_collector/config.py` | 采集参数（画布尺寸、圆圈大小、距离约束、透明度等） |
| `data_collector/collector.py` | 主采集程序 |

### 数据格式

```json
{
    "start_pos": [x, y],
    "end_pos": [x, y],
    "target_pos": [x, y],
    "trajectory": [[x, y, t_ms], ...],
    "total_duration_ms": 1234,
    "point_count": 150,
    "canvas_size": [942, 621],
    "collected_at": "2026-06-17T..."
}
```

### 碰到的问题

#### 问题 1: 全屏遮罩方案不可用

**现象**: v1 使用 tkinter `-fullscreen` + `-alpha 0.75` + 黑色背景 → 窗口全黑不透明，桌面完全看不到。

**原因**: `-alpha` 只是调节整窗透明度，不等于"透明 overlay"。黑色背景 × 0.75 透明度 = 暗色遮罩。

**尝试**: v2 改用 `-transparentcolor` + Win32 `WS_EX_TRANSPARENT`（鼠标穿透）→ 窗口直接不可见。

**最终方案**: v3 放弃全屏桌面叠加，改为**普通窗口 + 内部画布交互**。所有起终点圆圈在窗口画布内显示，点击由 tkinter canvas 统一处理。窗口尺寸 950×700，画布 `fill=both, expand=True`。

#### 问题 2: Canvas tag binding 点击不触发

**现象**: v2 中通过 `canvas.tag_bind("start_circle", "<Button-1>", ...)` 绑定圆圈点击，但点击后终点不出现。

**原因**: 每帧 `canvas.delete("all")` 后重建圆圈，tag binding 理论上应存活但实际不稳定。

**解决**: v3 改为统一的 `<Button-1>` 绑定在整个 canvas 上，在 `_on_canvas_click` 中手动计算点击位置与圆圈的距离（容差 35px），根据当前 `phase` 判断触发哪个逻辑。

#### 问题 3: pynput 坐标需转换

**现象**: pynput `on_move` 返回屏幕绝对坐标，但轨迹数据需要画布相对坐标。

**解决**: `canvas_x = screen_x - canvas.winfo_rootx()`，同理 y。录制时存入画布坐标，保证数据与窗口位置无关。

## Step 2: 数据预处理

### 计划

加载 JSONL → 坐标归一化（除以 canvas 尺寸）→ 转为相对位移 delta 序列 → 统一长度（pad/trunc 到 seq_len=200）→ PyTorch Dataset。

### 文件

| 文件 | 用途 |
|------|------|
| `training/config.py` | 训练超参数（seq_len, batch_size, hidden_size 等） |
| `training/dataset.py` | 数据加载 + 预处理 + DataLoader 划分 |

### 碰到的问题

#### 问题 4: 截断时 delta 累加和不等于原始位移

**现象**: 过拟合测试（8 条数据 200 epoch）Loss 降到 0.000058，但终点误差仍有 ~150px。

**原因**: 当原始轨迹点数 T > seq_len(200) 时，v1 代码直接对 delta 序列做均匀采样截断：
```python
indices = np.linspace(0, T-1, seq_len, dtype=int)
deltas = deltas[indices]  # BUG: 截断后 Σdeltas ≠ 原始 Σdeltas
```
取子集的 delta 累加和 ≠ 完整序列的累加和，导致模型学的是"不完整轨迹"。

**解决**: 在**绝对坐标**上重采样，再重新计算 delta：
```python
# 1. 还原绝对坐标
abs_pos = cumsum(deltas)
# 2. 在绝对坐标上均匀插值到 seq_len+1 个点
new_indices = np.linspace(0, T, seq_len + 1)
new_abs = np.interp(new_indices, np.arange(T+1), abs_pos)
# 3. 重新计算 delta
deltas = np.diff(new_abs, axis=0)
```
修复后截断轨迹仍保持完整位移信息。

## Step 3: LSTM 模型训练

### 计划

输入条件向量 `[start_x, start_y, end_x, end_y]` → 条件嵌入 → LSTM(2层, hidden=128) → 输出 `(dx, dy)` 序列。

训练用 Teacher Forcing（每步喂真实 delta），推理用 Autoregressive（喂自己预测的 delta）。

### 文件

| 文件 | 用途 |
|------|------|
| `training/model.py` | LSTM 模型定义（训练/推理双模式） |
| `training/train.py` | 训练循环 + 早停 + 模型保存 |

### 模型架构

```
输入 (4): [start_x, start_y, end_x, end_y]
    │
    ▼
Linear(4,32) → ReLU → Linear(32,32) → ReLU   ← 条件嵌入
    │
    ▼
LSTM(input=2+32, hidden=128, 2 layers, dropout=0.2)
    │   ↑
    │   每步输入: [prev_dx, prev_dy] + condition_embedding
    ▼
Linear(128,64) → ReLU → Linear(64,2)           ← 输出头 → (dx, dy)
```

### 训练超参数

| 参数 | 值 |
|------|-----|
| seq_len | 200 |
| hidden_size | 128 |
| num_layers | 2 |
| dropout | 0.2 |
| batch_size | 16 |
| lr | 1e-3 (Adam, ReduceLROnPlateau) |
| early_stop | 25 轮 |
| 训练/验证/测试 | 70/15/15 |

### 碰到的问题

#### 问题 5: PyTorch 2.12 DLL 加载失败

**现象**: `import torch` 报 `[WinError 1114] 动态链接库(DLL)初始化例程失败`，`c10.dll` 加载失败。

**原因**: PyTorch 2.12 太新，与 Python 3.12.0rc3 或系统缺少 VC++ Redistributable 不兼容。

**解决**: 降级到 PyTorch 2.5.1+cpu，正常导入。

#### 问题 6: 数据量不足导致训练发散

**现象**: 33 条数据训练后，生成轨迹终点偏离 1200+px（画布宽度仅 942px），模型在自回归模式下严重发散。

**原因**: 
1. 训练数据太少（33 条），模型学不到泛化模式
2. 自回归生成时小误差逐步累积，最终放大到几百像素

**过拟合验证**: 用 8 条数据训 200 epoch，Loss 降到 0.000058 但终点误差仍有 ~150px。说明：
- 模型容量足够（能记住训练数据）
- 自回归误差累积是结构性问题，需要更多数据 + 可能的架构改进

**建议**: 采集 200-500 条轨迹后重新训练；后续可考虑加入 scheduled sampling 或 endpoint guidance loss。

## Step 4: 推理验证

### 文件

| 文件 | 用途 |
|------|------|
| `demo.py` | 可视化演示（画布点击 → 生成轨迹动画） |

### demo.py 功能

1. 加载训练好的模型
2. 画布点击设置起点（绿）→ 点击设置终点（红）
3. 模型生成轨迹 → 蓝色动画播放
4. 终点校正：线性拉伸使轨迹终点精确落在目标上
5. 连续模式：上一终点自动变为下一轮起点

### 2026-06-17: demo.py 交互简化

**调整**: 取消工具栏/按钮，改为纯画布点击交互。

**原因**: 原始设计有「开始采集」按钮和绿色起点圆圈需要精准点击，步骤太多。

**新方案**:
- 第一下点击画布任意位置 → 起点（绿点）
- 第二下点击画布任意位置 → 终点（红点）→ 自动生成轨迹动画
- 第三下 → 上一终点变新起点 → 循环
- 右键重置，ESC 退出

### 2026-06-17: 推理阶段增加 Catmull-Rom 样条插值提升流畅度

**问题**: 模型输出固定 200 点，动画不够流畅。是否需要在训练时提高 seq_len？

**结论**: 不需要。LSTM 学的是轨迹形状（方向、曲率），不是打点密度。流畅度在推理阶段用插值解决。

**方案**: 模型输出 200 点 → Catmull-Rom 样条插值 → 800 点（4x），纯数学计算，无需重新训练。

**对比**:

| | 训练时提高 seq_len | 推理时插值 |
|------|------|------|
| 训练时间 | 显著增加 (2-4x) | 零 |
| 需要重新训练 | 是 | 否 |
| 收敛难度 | 增加 | 不影响 |
| 轨迹形状质量 | 需更多数据支撑 | 完全不改变形状 |
| 流畅度提升 | ✅ | ✅ |

**选择**: 推理时 Catmull-Rom 样条插值（不依赖 scipy，纯 numpy 实现）。`SMOOTH_FACTOR=4` → 200 点 → 800 点。
